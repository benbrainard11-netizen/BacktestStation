## What to click in the morning

### 1. Strategy-aware Run-a-Backtest
- Visit **`/backtests/new`**.
- The "Strategy" dropdown now lists **Moving Average Crossover** + **Fractal AMD** (was hardcoded to MA-only before).
- Pick **Fractal AMD**. The "Strategy params" section renders typed fields: `max_risk_dollars`, `max_risk_pts`, `min_risk_pts`, `target_r`, `max_trades_per_day`, etc., each pulled from `FractalAMDConfig` defaults.
- Need raw JSON? Hit "Switch to raw JSON →" — same data, editable as text.

### 2. Risk Profiles
- Visit **`/risk-profiles`**.
- You'll see 3 auto-seeded profiles:
  - **Conservative**: $300 cap, 1 trade/day, 2R target. Daily loss cap 2R, DD cap 5R.
  - **Live-mirror**: $300 cap, 2 trades/day, 3R target. Daily loss cap 4R, DD cap 10R. ← the one matching your live bot.
  - **Aggressive**: $500 cap, 3 trades/day, 3R target. Daily loss cap 8R, DD cap 20R.
- Click any name to edit. You can rename, delete, or change params freely — the seed only fires once per name.
- Back at **`/backtests/new`**, when at least one profile exists, a "Risk profile" dropdown appears. Selecting Live-mirror prefills the strategy params with `max_risk_dollars=300, max_trades_per_day=2, target_r=3`.

### 3. Chart replay
- Visit **`/replay`** directly, or go to any backtest run dossier (`/backtests/{id}`) and click the new **▶ Chart replay** button.
- Form: pick **symbol**, **date**, and optionally **a backtest run** to overlay.
- Hit **Load** → 1m candles render, plus arrows on entry bars (▲ green for long, ▼ red for short) with PnL in the tooltip.
- **Playback controls:**
  - ⏮ resets to first bar.
  - ▶/⏸ plays/pauses.
  - ⏭ steps one bar.
  - **Speed buttons** (1x / 5x / 30x / 60x) change tick rate.
  - Scrubber lets you jump anywhere in the day.
- Below the chart, the "Entries this day" list grays-out future entries and lights up as the cursor passes them.
- **Default date is 2026-04-22** which has live trades and a recent port-FILLED run — quickest way to see the overlay. Pick run id=10 (the MNQ-downshift smoke) to see entry markers on bars.
- **Empty state**: if the warehouse doesn't have bars for the picked date, you'll get a "No bars in the warehouse" panel instead of a chart.

## What this *isn't* (deferred — your call when to come back to it)

- **Tick-level (TBBO) granularity in replay.** The MVP renders 1m candles, not raw ticks. The infra to load TBBO is already there (`read_tbbo()` in `app/data/reader.py`); next session can add a streaming endpoint and a tick-by-tick visual.
- **FVG-zone / SMT divergence overlays on the chart.** The `TracingFractalAMD` subclass already records every setup, touch, and rejection per bar; the next replay-v2 chunk would expose those events as a JSON timeline and draw them as colored boxes on the chart.

- **Strategy registry refactor.** Per CLAUDE.md, premature with 2 strategies. The current hand-maintained `STRATEGY_DEFINITIONS` mirrors the engine resolver and is fine.
- **Risk-profile rule-violation visualization** in the dossier. The evaluator endpoint exists; a "what would Live-mirror have flagged?" panel is a small frontend follow-up.
